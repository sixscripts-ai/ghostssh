System Architecture: Appwrite + Playwright Worker

How should the Appwrite connections be defined?

A

Option A: API-Centric (Recommended)

Only the Fastify API uses the Appwrite Server SDK. The Dashboard and Playwright Worker query the Fastify API.

[Next.js Dashboard UI]
  ↓ (HTTP GET / POST)
[Fastify API] → 🅰️ Appwrite DB
  ↑ (HTTP GET / POST)
[Playwright Apply Worker]
Pros:
  • API remains the single source of truth for all DB logic
  • Playwright worker can be a simple script anywhere
Cons:
  • Slightly more API routes to write in Fastify
B

Option B: Direct Web/Server SDKs

Dashboard uses Appwrite Web SDK directly. Fastify API and Playwright Worker use Server SDK directly.

[Next.js Dashboard] ---┐
[Fastify API]        ├─→ 🅰️ Appwrite DB
[Playwright Worker] ---┘
Pros:
  • Next.js dashboard gets direct real-time DB subscriptions
Cons:
  • Logic is duplicated (Dashboard fetches jobs directly, API also fetches jobs)
  • We'd need to set up Appwrite Client SDK in Next.js
